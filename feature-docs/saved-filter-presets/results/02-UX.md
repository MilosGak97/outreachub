# UX Pilot Prompt Pack — Saved Filter Presets (Filters-only)

This file contains **copy/paste prompts specifically for UX Pilot (Figma plugin)**.  
Use them in order. Each prompt is designed to generate **frames + components** without drifting into API/DB.

> **Feature:** Saved Filter Presets (filters-only)  
> **Surfaces:** Admin + Client  
> **Pages:** Static list pages (e.g., Admins, Companies) + Dynamic object list pages  
> **Scope Key:** `conceptKey + tableId` (preset is scoped to this, plus user + company/tenant)

---

## Global UX Rules (apply to every prompt)
- **Filters-only.** Do NOT include: column visibility/order, sorting, table layout.
- Preset control lives **INSIDE the Filter Panel** (not top bar).
- Save/Overwrite uses **MODALS**.
- After applying a preset, if user changes any filter/search → show **CUSTOM** state with actions: **Save As** and **Overwrite**.
- Missing/changed dynamic fields while applying preset: **ignore silently** (no toast).
- Default preset auto-applies on page open; clearing default returns to **normal view (no filters selected)**.
- URL sharing uses **presetId only** (no raw filter params).

# Prompt 1 — Create Page-Level Frames (List Page + Filter Panel)

**Paste into UX Pilot:**

Create UX frames for a CRM SaaS List Page that includes a Filter Panel with a “Saved Filter Presets” section.

Frames to create (desktop):
1) List Page — No presets yet (empty presets state)
2) List Page — Presets exist (none selected)
3) List Page — Default preset exists (auto-applied on load)
4) List Page — Preset selected (filters reflect preset)
5) List Page — Custom state (user changed filters after preset applied)

Requirements:
- Preset selector is inside the Filter Panel.
- Include a compact preset dropdown/list area with actions.
- Show “Default” badge on the default preset.
- Show “Custom” badge/state when filters changed.
- Include table area (simple placeholder) but focus on Filter Panel behavior.
- Use clean enterprise CRM styling.

Do NOT include columns/sort/layout features.

# Prompt 2 — Filter Panel Components + Interaction Detail

**Paste into UX Pilot:**

Design the Filter Panel UI components for “Saved Filter Presets” (filters-only).

Include:
- Preset dropdown/list (shows preset names)
- Actions: Apply/Select preset, Save preset, Save As, Overwrite, Delete, Make Default, Clear Default, Reset Filters
- States: 
  - empty (no presets)
  - preset selected
  - default preset selected
  - custom state after modifications

Interaction rules to reflect in UI:
- When preset selected → filter controls update + table reload (show loading state)
- When user changes any filter/search after preset → state becomes “Custom” with Save As / Overwrite options
- Clear default returns to normal view (no filters selected)

Do NOT add columns/sorting controls.

# Prompt 3 — Modal Designs (Save / Save As / Overwrite / Delete)

**Paste into UX Pilot:**

Create modal designs for Saved Filter Presets.

Modals:
1) Save Preset
   - fields: Preset Name (text)
   - checkbox: Set as default
   - buttons: Cancel / Save
   - validation: name required; name unique per context; max 50 presets per context
2) Save As Preset
   - same as Save Preset
3) Overwrite Preset
   - shows current preset name (editable)
   - copy explains it updates: name + current filters
   - buttons: Cancel / Overwrite
4) Delete Preset confirmation
   - warning text
   - buttons: Cancel / Delete

Note:
- Overwrite updates BOTH name + filters.
- Keep enterprise tone, clear microcopy.
