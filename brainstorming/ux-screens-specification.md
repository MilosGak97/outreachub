# UX Screens Specification for UX Pilot

> **Purpose**: Detailed screen-by-screen specifications for Figma UX Pilot plugin to generate admin and client UI for the CRM Template System.

---

## Design System Notes

**Style Guide**:
- Clean, modern SaaS admin interface
- Neutral color palette with accent color for actions
- Card-based layouts for list items
- Consistent spacing (8px grid)
- Clear visual hierarchy
- Accessible contrast ratios

**Common Components**:
- Breadcrumbs for navigation
- Search inputs with icon
- Pagination controls
- Action buttons (primary, secondary, danger)
- Status badges/pills
- Empty states with illustrations
- Loading skeletons
- Toast notifications
- Confirmation modals

**Elements**:

1. **Page Header**
   - Breadcrumb navigation
   - Page title "CRM Templates"
   - Subtitle with description
   - Primary action button "+ Create Template" (top right)

2. **Filters Bar**
   - Search input with magnifying glass icon
   - Status dropdown filter (All, Active, Draft/Inactive)
   - Results count text

3. **Templates Grid**
   - Card-based layout, 3 columns on desktop
   - Each card shows:
     - Icon (large, centered at top)
     - Template name (bold)
     - Description (2 lines, truncated)
     - Stats: "X modules" and "Y companies using"
     - Status badge (Active = green, Draft = gray)
     - Three-dot menu for actions (Edit, Duplicate, Delete)
   - Special "Create New" card with plus icon

4. **Pagination**
   - Standard pagination controls
   - Page numbers with prev/next

**States**:
- **Loading**: Skeleton cards
- **Empty**: Illustration + "No templates yet" + Create button
- **Error**: Error message with retry button

**Interactions**:
- Click card â†’ Navigate to template detail
- Click "..." menu â†’ Show dropdown (Edit, Duplicate, Delete)
- Click "+ Create Template" â†’ Navigate to create form
- Search â†’ Filter cards in real-time
- Status filter â†’ Filter by active/inactive
